# style-clone — 한국어 문체 지문 복제·진단 엔진 & 에이전트 스킬

> **문체를 서술의 대상에서 측정과 재현의 대상으로.**
> "간결하고 시니컬한 문체로 써줘" 대신 — 작가의 글에서 **측정 가능한 지문**을 뽑고, 그 지문으로 생성을 **조건화·채점·교정**한다.

LLM에게 "이 글처럼 써줘"라고 원문을 통째로 주면 두 가지 일이 벌어진다. 원문 표현을 그대로 **복사**하거나(주제가 바뀌면 무너진다), 어디서나 본 듯한 **평균적인 글**로 뭉개진다(LLM 평탄화 — 문헌으로 실증된 현상이다). 이 저장소는 세 번째 길을 구현한다:

```
작가 코퍼스 → [지문 추출] → 지문 JSON → [지문 조건 생성 ×N] → [문단 재순위화]
                                              ↑                      ↓
                                        [교정 처방] ←── [채점: 수용·평탄화·캐리커처]
```

블라인드 LLM 심판 실험에서 이 파이프라인은 "원문 통째 모사" 방식을 **2:0**으로 이겼다.

## 무엇이 들어 있나

| 구성 | 설명 |
|---|---|
| **파이썬 엔진** (의존성: 표준 라이브러리, 선택적으로 kiwipiepy) | 지문 추출·채점·재순위화·교정 처방·종결어미 변환 |
| **에이전트 스킬** (`skill/`) | Claude Code·Codex가 위 엔진을 조종해 문체 복제/진단을 수행하는 지시서 |
| **연구 문서** (`docs/`) | 9층 문체 분류체계, 측정 수식, 문헌 근거, 설계 감사 |
| **벤치마크** | 1930년대 작가 5인(김유정·이상·현진건·채만식·이효석) 판별 과제 + 회귀 테스트 |

## 핵심 아이디어 4개

**1. 지문 = 분포 + 분산.** 종결어미·연결어미·조사 분포, 문장 길이 리듬, 담화 구조(의미 도약, 무표지 연결, 수미상관), 그리고 각 특징의 **작가 내 변동폭**까지 저장한다. 재현할 것은 표현이 아니라 습관의 분포다.

**2. z와 d의 분리.** 채점은 두 통계량을 절대 섞지 않는다.
- **z (수용 판정)**: 작가 자신의 변동폭으로 정규화한 이탈도. z ≤ 1이면 "작가의 정상 범위 안".
- **d (절대 거리)**: 후보 줄 세우기(재순위화)와 작가 판별용.

**3. 캐리커처 가드.** 지문에 맞춰 최적화하다 보면 "작가보다 더 작가다운" 위조가 나온다. 후보의 d가 작가 본인 미공개(held-out) 글의 하한보다 **낮으면** 개선이 아니라 위조 심화 — 즉시 정지한다.

**4. 산포 비대칭 벌점.** instruction-tuned LLM은 체계적으로 문장을 균질화한다(ICLR 2024 등). 그래서 산포(길이 변화, 어휘 다양성)가 작가보다 **부족한 쪽에만** α=1.5배 벌점을 준다. 실제로 자체 실험의 1차 우승 후보가 이 감사에 걸렸다(길이 CV 0.34 vs 작가 0.53).

## 빠른 시작 (파이썬)

```bash
git clone https://github.com/pandoll-ai/style-clone-skill && cd style-clone-skill
python3 -m venv .venv && .venv/bin/pip install kiwipiepy pytest   # kiwipiepy 없어도 표층 모드로 동작

# 1) 지문 추출 (대조 코퍼스를 주면 '이 작가가 안 쓰는 표지' 금지 목록이 생긴다)
mkdir -p fingerprints
.venv/bin/python stylometry.py build corpus/yisang/날개.txt yisang corpus/kimyj/봄봄.txt

# 2) 채점 — z(수용)·d(거리)·금지 위반·평탄화 진단
.venv/bin/python stylometry.py score fingerprints/yisang.json 내글.txt

# 3) 후보 N개 재순위화 + 3중 검증 (수용·평탄화·캐리커처)
.venv/bin/python rerank.py fingerprints/yisang.json out.txt 후보1.txt 후보2.txt --baseline=작가_다른작품.txt

# 4) 교정 처방 — LLM에게 그대로 줄 수 있는 방향 있는 지시문
.venv/bin/python loop.py fingerprints/yisang.json out.txt --baseline=작가_다른작품.txt

# 5) 종결어미 변환 (합쇼체·한다체·해요체, kiwipiepy 필요)
.venv/bin/python postprocess.py hapsyo 글.txt

# 벤치마크·테스트
.venv/bin/python benchmark.py            # 작가 5인 held-out 판별 + 부트스트랩 CI
.venv/bin/python -m pytest tests/ -q
```

`loop.py`가 내놓는 처방은 이런 식이다 — 사람도 LLM도 바로 실행할 수 있다:

```
[z=99]  연결어미 '니까'는 이 작가가 쓰지 않는 표지다 — 전부 제거·치환하라
[z=2.2] 평균 문장 길이(어절): 현재 4.05, 목표 6.03 — 문장을 더 길게 이어 붙여라
[z=1.9] 문장 길이 산포(CV): 현재 0.37, 목표 0.54 — 단문과 장문의 낙차를 키워라 (과평탄)
```

## 에이전트 스킬로 쓰기 (Claude Code / Codex)

```bash
bash skill/sync.sh    # 엔진을 skill/scripts/로 동기화하고 ~/.claude/skills/style-clone에 설치
```

이후 Claude Code에서 "**이상 문체로 하네스를 설명하는 에세이 써줘**" 같은 요청이 오면 스킬이 발동해:
코퍼스 75/25 분할 → 지문 추출 → 지문을 생성 조건으로 번역 → 후보 4개+ 생성 → 문단 재순위화 → 교정 루프(최대 2회전) → **채점표와 함께** 최종 글을 내놓는다. 점수가 나쁘면 나쁘다고 보고하는 것까지가 스킬의 계약이다.

Codex는 `~/.codex/prompts/style-clone.md` 래퍼로 같은 지시서를 공유한다.

두 가지 모드:
- **clone**: 코퍼스 + 주제 → 그 문체의 새 글 (+ 채점표)
- **audit**: 글 + 코퍼스 → 문체 이탈 정량 진단 리포트 (어느 층위가 얼마나 다른지, 무엇을 고칠지)

## 검증된 결과

| 실험 | 결과 |
|---|---|
| 저자 판별 (김유정 vs 이상, held-out) | 측정층 89%, 문자 bigram 100% |
| 저자 판별 (동시대·동장르 5인) | 30문장 단위 75%, bigram 90% — 판별 단위가 클수록 정확 (Eder 2015 부합) |
| 생성 대결 (지문 조건 vs 원문 통째 모사) | 블라인드 LLM 심판 2:0 승. 베이스라인은 금지 표지 3건 + z 6.35로 수용 영역 밖 |
| 교정 루프 수렴 | z 6.35 → 0.15 (2회전). 스킬 E2E에서도 z 3.0 → 0.67 (1회전) |
| 평탄화 자동 검출 | 1차 우승 후보의 과평탄(len_cv 0.34<0.53)을 산포 벌점이 적발 |
| 캐리커처 가드 발동 | 지문에 직접 최적화한 조합(d 0.246 < 작가 하한 0.431)을 정지시킴 |

정직 노트: **kiwipiepy 형태소 교체는 판별 정확도를 유의하게 올리지 못했다** (신뢰구간 겹침). 그래도 유지하는 이유는 이형태 수렴(`모른다/자란다 → -ㄴ다`)이 지문의 해석성을 올리고, 종결어미 변환이 EF 태그를 필요로 하기 때문이다. 이 프로젝트는 "새 특징은 아이디어가 아니라 실적으로 채용된다"는 관문을 스스로에게도 적용한다.

## 저장소 구조

```
stylometry.py            지문 추출·채점 엔진 (분석기 이중화 + 담화층 통합)
features_surface.py      L1 표층 접미 특징 — 의존성 없는 폴백
features_morph.py        L1 kiwipiepy 형태소 태그 특징
discourse.py             담화·논리 층 특징 (의미 도약, 리듬, 수미상관, 산포)
rerank.py                문단 단위 best-of-N + 3중 검증
loop.py                  교정 루프: 진단 → 방향 있는 처방
postprocess.py           종결어미 스피치 레벨 결정론적 변환
benchmark.py             작가 5인 표준 벤치마크 (부트스트랩 CI)
validate.py              v1 회귀 기준선 (2인 판별)
fetch_wikisource.py      위키문헌 코퍼스 수집기
corpus/<작가>/           1930년대 작가 5인, 저작권 만료 작품 (corpus/SOURCES.md)
fingerprints/            추출된 지문 JSON 예시
docs/                    연구 문서 (아래)
skill/                   배포용 에이전트 스킬
tests/                   회귀 테스트 (pytest)
```

## 한계

- 채점 대상이 30문장 미만이면 신뢰도가 낮다 (실측 기반 — 출력에 `low_confidence` 플래그).
- 내용 충실도(주제가 유지됐는가)는 아직 측정하지 않는다 — 블라인드 LLM 심판이 임시 대역 (로드맵 P4).
- 담화층 프록시는 어휘 중첩 기반이라 동의어 치환에 둔감하다 (로드맵 P2에서 임베딩으로 해소 예정).

## 데이터 고지

- `corpus/`의 문학 작품은 전부 **저작권 만료**(저자 사후 70년 경과) 공개 저작물이다. 출처·수집 방법은 `corpus/SOURCES.md`.
- 원 연구의 생성 대결에 쓰인 **대상 에세이와 생성 후보들은 저자 개인 글이라 이 공개 저장소에서 제외**했다. 해당 파일을 쓰는 회귀 테스트는 자동으로 skip 된다 (수치 기록은 REPORT.md에 보존).
