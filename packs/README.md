# 모델 스타일 팩 (Model Style Packs)

작가 지문이 "이 사람처럼 쓰기"라면, 스타일 팩은 그 반대 방향이다 — **모델 특유의
문체를 측정하고 제거**하기 위한 서명 사전. 모델이 대량 생성한 코퍼스에서 인간
대조군 대비 과하게 나타나는 특징을, 판별 실적 관문(|Cohen's d| ≥ 0.5)을 통과한
것만 골라 고정한다.

핵심 이점: **팩은 만인의 대조군**이다. 작가 지문의 금지 목록은 대조 코퍼스가
매번 필요하지만, 스타일 팩은 한 번 만들면 어떤 글에도 "이 글에 코덱스 티가 얼마나
묻었나"를 대조군 없이 물을 수 있다.

## 사용

```bash
python3 style_pack.py score packs/codex-style.json 글.txt   # 코덱스니스 점수 + 마커
python3 style_pack.py strip packs/codex-style.json 글.txt   # 제거 처방 (LLM 지시문)
```

## codex-style.json (파일럿, 2026-07-11)

- **생성**: gpt-5.6-sol (codex-cli 0.144.1, effort=low), 3장르(에세이·설명문·백과) ×
  12주제 = 36편. 스타일 지시어 없는 최소 프롬프트로 프롬프트 지문 오염 최소화.
- **대조군**: 같은 주제의 한국어 위키백과 12편 (인간 산문).
- **판별력**: 코덱스 vs 인간 15문장 청크 판별 **89.8%** (CI [84.3, 94.5], 52특징 벡터).
- **검증**: 홀드아웃 코덱스 에세이 점수 16.76 vs 인간 위키 1.76 — 명확 분리.

### 추출된 코덱스 서명

| 층위 | 코덱스 특징 |
|---|---|
| 요인 | 길이 균질성 ↑↑ (d=+1.91, 평탄), 주제 반복성 ↓ (d=-1.45), 결속 암시성 ↑ |
| 구문 | "~이 아니라 ~이다" 대조 구문(인간의 3.9배), 관형절 연쇄 |
| 종결어미 | -는다·-진다·-하다 편중 (설명체 현재형) |
| 접속부사 | **물론·그래서·역시·결국** 과다 (물론은 인간 0% → 코덱스 4.4%) |
| 연결어미 | -지만·-면서 과다 |
| 화용 | 1인칭 노출·단정 강조어 |

가장 강한 단일 신호는 **접속부사 남발**(물론/그래서/역시/결국)과 **길이 균질성**
(문장을 고르게 만드는 평탄화) — 문헌의 LLM 평탄화 실증(README §핵심 4)과 정확히 일치.

### 한계 (정직 고지)

- 표기 계열(로마자·한자·숫자·괄호)은 위키 장르가 코덱스 에세이보다 압도적으로 많아
  **장르 교란**이다 — 서명에서 제외했다. 순수 문체 마커만 채택.
- 대조군이 위키 한 종이라 "코덱스 vs 인간"이 아니라 "코덱스 산문 vs 위키체"에 가깝다.
  다장르 인간 대조군(뉴스·블로그·소설)으로 확장하면 장르 교란이 더 빠진다.
- 요인 공간은 1930년대 소설 코퍼스에서 추정 — 현대문 절대값은 왜곡 가능.
- 모델 버전 고정: 이 팩은 gpt-5.6-sol 전용. 다른 모델·버전은 별도 팩.

### 재현

`meta.recipe` 참조. `corpus_samples/`에 장르별 원문 1편씩(전체 코퍼스는 재현
가능하므로 미배포). 확장·재추정은 `factor_analysis.py`와 동일 파이프라인.

## 로드맵

- **claude-style 팩**: sonnet 5 서브에이전트로 동일 프로토콜 → 클로드 티 제거·강화용.
- **장르 팩**: 공문서·소설·강의안 등 목표 레지스터 (제거가 아니라 부여 방향).
- **개인 팩**: 작가 지문과 같은 형식 — gn-writing-style이 첫 사례.
- 오픈 데이터: 팩 규격(요인 서명 + 마커 + 분포)만 배포, 원문 코퍼스는 레시피로.
