# P1.5 — 요인 구조 (한국어 문체 Big-N) 실행 계획

2026-07-11 착수. Biber MDA(1988)의 방법론을 한국어에 이식한다: 원자 특징을 확장하고,
요인은 설계하지 않고 **데이터에서 추출**하며, 요인별 신뢰도·판별력 관문을 통과한 것만
채점기에 편입한다.

## 목표 도달 기준 (Goal Gates — 전부 수치로 판정)

| # | 기준 | 판정 |
|---|---|---|
| G1 | 원자 스칼라 특징 ≥ 55개 (L1·L4-L9 기존 ~30 + L2/L3/L7/구문 신규 ≥ 25), 전부 kiwi 이하 티어 | 특징 목록 카운트 |
| G2 | 요인 추출: 요인 수는 평행 분석(parallel analysis)이 결정, 각 요인은 QG3 관문 통과 | factors.json 산출 |
| G3 | 요인 기반 판별이 기존 원자 판별 대비 **유의하게 하락하지 않음** (30문장, CI 겹침 허용). 15문장에서 개선되면 초과 달성 | benchmark 비교 |
| G4 | 스키마 v3(원자+요인) 통합 후 회귀 테스트 전원 통과 (v1 25/28, C2 검출 유지) | pytest |

미달 시: 부분 성과라도 정직하게 기록하고 커밋 (숨기지 않음). G3 실패면 요인은
'진단 전용'(채점 미편입)으로 강등하고 실패 원인을 §회고에 기록.

## 아키텍처 결정

- **런타임 엔진은 표준 라이브러리 유지** (스킬 경량 제약). 요인 분석(고유분해·회전)은
  dev 도구 `factor_analysis.py`에서 numpy로 수행하고, 산출물(표준화 파라미터+로딩 행렬)을
  `factors.json`으로 고정 → 채점 시에는 순수 파이썬 행렬곱만.
- [ASSUMPTION] 요인 공간의 표준화·로딩은 1930년대 소설 5인 코퍼스에서 추정 — 현대 산문
  일반화는 미검증. 영향: 현대문 요인 점수의 절대값 왜곡 가능. R3 회고에서 현대 텍스트로 점검.

## 단계 및 품질 게이트

### S1. 원자 특징 확장 (`features_ext.py`)
- L2 통사(kiwi): 명사화 밀도(ETN+'것'), 관형형(ETM) 밀도, 피동·사동(XSV) 밀도,
  주어 생략률(주격 표지 부재 근사), 용언 밀도(아이디어 밀도 T1 근사), 내포 근사(문장당 ETM).
- L3 어휘: 추상도 접미(-성/-화/-도/-론/-적) 밀도, 감각어 사전 밀도, 구어 표지,
  외래어(로마자/외래 표기) 밀도. ※ 고유어/한자어 어종 구분은 사전 부재로 보류 — 감사문서에 기록.
- L7 화용: hedge/booster 사전 비율, 1인칭 노출도, 독자 호명(2인칭+청유·명령), 간접인용 밀도.
- 구문(constructions): "A가 아니라 B" 패턴, "~할 수 있-" 밀도, "것이다" 종결률,
  삼항 병렬, 문두 총괄 표지(결론적으로/요컨대), 괄호·콜론 사용.
- **QG1**: 특징별 (a) 5작가 코퍼스에서 분산>0 (degenerate 기각), (b) 특징 간 |r|>0.95 중복
  병합, (c) 단위 테스트. → 통과 목록을 R1 회고에 기록.

### S2. 특징 행렬 + 요인 추출 (`factor_analysis.py`, dev 전용)
- 5작가 전 청크(30문장 단위, ~90개) × 전체 스칼라 행렬 → 표준화 → 상관 → PCA+varimax.
- 요인 수 = 평행 분석 (무작위 행렬 고유값 95분위 초과분).
- **QG2**: 표본 적정성(청크 수 ≥ 특징 수 경고 명시), 재현 스크립트 커밋.

### S3. 요인 관문 심사
- **QG3** — 요인별 3중 관문:
  (a) 신뢰도: 작가내/작가간 분산비 F ≥ 2 (청크 안정성),
  (b) 판별 기여: 요인 점수만으로 5인 판별 ≥ 우연(20%)의 2배 or 제거 시 판별 하락,
  (c) 해석 가능성: 상위 로딩 특징이 의미적으로 묶여 **이름을 붙일 수 있음**.
- 통과 요인만 `factors.json` 채용. 탈락 요인은 사유와 함께 기록.

### S4. 엔진 통합 (스키마 v3)
- `build_fingerprint`: 요인 점수의 작가 청크 mean/std 저장 (`fp['factors']`).
- `score_text`: 후보 요인 z (순수 파이썬), 진단에 요인 이름 표기. loop.py 처방에 요인 층 추가.
- **QG4**: 기존 회귀 15종 + 신규 테스트 통과. v1 수치 불변(요인은 additive).

### S5. 검증·회고·배포
- benchmark: 원자 vs 요인 vs 병행, 15/30문장 (G3 판정).
- 감사문서(05)·README 갱신, skill sync, 커밋.

## 회고 기록

### R1 (S1 후, 2026-07-11) — 특징 품질

- 신규 27특징 전원 비퇴화. 판별 신호 강함: 1인칭 노출 F=81.5, 관형형 밀도 F=47.4, 피동·사동 F=40.3.
- **기각/제외 4건** (모듈에는 유지, 요인 입력에서만 제외 — 모델 팩 전용 후보):
  `CX_meta_discourse`(유병 1.1% — 1930s 소설에 없는 AI 표지), `CX_triple_list`(F<1),
  `L7_quote_particle`(유병 4.7%), `CX_paren_colon`(한자 병기와 r=0.956 중복).
- `B_jump_p90`은 30문장 청크에서 1.0 포화 퇴화 → 자동 기각 (측정명세 §한계 예측 적중).
- 어종(고유어/한자어) 구분은 사전 부재로 보류 — kiwipiepy는 어원 정보를 주지 않음.

### R2 (S2-S3 후) — 요인 구조

- 행렬 277청크 × 51특징 (관측/특징 5.4 — 경계 수준, 코퍼스 확장 시 재추정 필요).
- 평행 분석 → **요인 8개**, 전원 신뢰도 관문 통과 (F비 6.4~98.0).
- 명명: 문장 호흡 / 길이 균질성 / 결속 암시성 / 주제 반복성 / 표기·대화 습관 /
  문어 격식성(F=98, 최강 판별축) / 장단 교대성 / 구체성. 8개 전부 이름이 붙음 — 관문 (c) 통과.
- **편입 수준 결정: 요인은 진단 층** — score_text의 z_total 산식에 미반영 (기존 회귀 보존).
  채점 통합은 요인 가중 z가 원자 z보다 나은지 벤치마크 근거 확보 후 별도 심사.

### R3 (S5 후) — 목표 판정표

| Gate | 기준 | 실측 | 판정 |
|---|---|---|---|
| G1 | 원자 스칼라 ≥55 | 57개 (기존 30 + 신규 27) | **PASS** |

| G3 | 요인 판별 하락 없음(30문장) | 요인 73.3% [67.5,78.3] vs 사전 기준 75.0% [64.7,85.3] — CI 겹침 | **PASS (문언상)** |

**G3 정직 주석 (프로토콜 통제 후 진실)**: 동일 LOO-중심점 프로토콜로 재면
원자 52특징 76.8%[73.4,80.4]@15문장 / 83.4%[78.3,87.4]@30문장 vs
요인 8차원 65.5%[61.8,69.7]@15 / 73.3%[67.5,78.3]@30 — **요인 압축은 판별 정보를 잃는다**
(CI 분리). 요인의 존재 이유는 판별이 아니라 ① 이름 있는 진단(C2의 결함이
'길이 균질성 +4.3, 장단 교대성 +9.2'로 표면화), ② 팩 설계의 좌표계, ③ 8차원 프로파일의
이식성이다. **이번 단계의 최대 보너스는 특징 확장 자체**: 15문장 판별이 기존 d 기반
53.6%에서 52특징 벡터 76.8%로 상승 — 짧은 텍스트 채점의 저신뢰 문제를 완화할
후속 과제(벡터 판별기의 benchmark.py 정식 편입)가 확인됨.
