# 연구 로드맵

## 특징 채택의 유일한 관문 (전 단계 공통)

새 특징은 아이디어가 아니라 실적으로 채용된다: **held-out 저자 판별 정확도를 유의하게 올리거나, 기존 특징이 못 잡는 실패 사례(평탄화·복사 등)를 검출해야 편입.** 채점기에 들어간 특징은 그 실험의 평가 패널에서 제외(Goodhart 규율). 모든 특징은 최소 표본 크기와 신뢰구간을 기록하고, CI가 넓으면 채점에서 자동 제외.

## P0 — 기반 교체 (1주 규모)

- kiwipiepy 도입: L1 특징을 표층 패턴 → EF/EC/J*/MAG 태그 기반으로 교체. 어종 사전 연결.
- 코퍼스 확장: 작가당 3작품+ (위키문헌·공유마당·다빈치맵). 장르 통제(같은 장르 내 판별로 난도 상승).
- 벤치마크 고정: 작가 5인 × held-out 판별을 표준 과제로 삼고 모든 특징 추가를 이 수치로 심사.

## P1 — 담화 층 정식 편입 (구현 일부 완료)

- `discourse.py`(T0 프록시) 특징을 지문 스키마에 통합, 청크 분산 포함.
- 산포 비대칭 벌점(과평탄 α=1.5) 구현 → C2류 평탄화 오버피팅 자동 검출 회귀 테스트.
- 위치 조건부(사분위) 분포 저장 — 기승전결 배분의 지문화.

## P2 — 임베딩 층 (T2)

- KoSimCSE 또는 BGE-M3로 semantic_jump, topic_drift_curve, 문단 응집도.
- 검증: T0 어휘 중첩 프록시와의 상관 확인 후 교체(상관 높으면 T0 유지가 이득 — 해석 가능성).

## P3 — 정보 궤적 층 (T3)

- Polyglot-Ko(1.3b면 충분)로 어절 surprisal 궤적: 평균, CV, lag 자기상관, 저주파 스펙트럼 비율.
- 가설 검증 실험: "필자별 surprisal 궤적 모양이 판별 특징이 된다" — 문헌(Harmonic Structure)이 언어 일반에서 주기성을 확인했으므로, 개인차가 판별력을 갖는지가 신규 기여 지점. **이게 이 프로젝트의 첫 논문감.**

## P4 — 추론 구조 층 (T4)

- klue-roberta NLI로 entailment_gap(추론 보폭 정밀판). '천재 스킵'의 조작적 정의: gap 상위 + 전역 일관성 유지 + 독자 판별기 통과.
- entity grid(개체 역할 전이), 스탠스/hedge 분류기.
- LLM 라벨링 경로(LISA 방식): 판별기에게 문체 차이를 언어로 설명시키고 → 설명을 특징 후보로 번역 → P0 벤치마크로 심사 (루프 3의 제도화).

## P5 — 학습형 하이브리드

- 수제 특징(처방 담당) + 대조학습 문체 임베딩(재현율 담당, StyleDistance 방식의 한국어판: LLM으로 합성 병렬 문체쌍 생성 → 대조학습).
- 재순위화는 하이브리드 점수, 교정 루프의 진단은 수제 특징만 사용(임베딩은 처방을 못 주므로).

## 상시 검증 프로토콜

1. 작가 판별 벤치마크 (P0 고정 과제) — 측정층의 회귀 테스트.
2. 생성 대결 (지문 조건 vs 원문 통째) — 시스템 수준 A/B, 블라인드 심판 n≥6 + 순서 교차.
3. 평탄화 감사 — 모든 승자 후보에 대해 L9 산포 지표를 작가 하한과 비교, 미달 시 루프 재가동.
4. 내용 충실도 — 양방향 NLI 게이트(P4 이후), 그 전에는 LLM 심판.
5. 캐리커처 가드 — d < 작가 본인 held-out 기준선이면 정지·경고 (v1에서 이미 발화 확인).
